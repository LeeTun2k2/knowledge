## gRPC


## Key Ideas

Binary base
HTTP/2
Protocol Buffer
Bi-direction streaming.
Low latency
Complex to use.
Build-in Load balancer.
Best for complex, high-performance system. Microserver

## Overview

Is a framework help transfer data, DO NOT depend on programming language.
Is design based on HTTP/2. 

## Purpose

Lightweight, high performance.
Contract first, use protocol buffer
Generate strongly typed server - client
Support bi-direction call
Reduce network with protobuf binary serialization.

## Usage

For internal microservice, require critical effeciency.
Realtime service that need to handle request and response.
